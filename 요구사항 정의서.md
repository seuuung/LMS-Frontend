# 📌 시스템 요구사항 정리

## 1. 관리자 페이지 (Admin)
> 관리자만 접근 가능

### 1-1. 유저 관리
- 유저 정보 **조회**
- 유저 정보 **추가**
- 유저 정보 **수정**
- 유저 정보 **삭제**

---


### 1-2. 클래스 관리
> 클래스 소유자 여부와 관계없이 관리 가능

- 클래스 **조회**
- 클래스 **검색**
- 클래스 **생성**
    - 클래스 소유자 지정 필수
- 클래스 **수정**
- 클래스 **삭제**

---

### 1-4. 강의 관리
> 클래스 소유자 여부와 관계없이 관리 가능

- 클래스 내 강의 **조회**
- 클래스 내 강의 **생성**
- 클래스 내 강의 **수정**
- 클래스 내 강의 **삭제**

---

### 1-5. QnA 게시판 관리
> 클래스 소유자 여부와 관계없이 관리 가능

- 게시글 **조회**
- 게시글 **생성**
- 게시글 **삭제**

---

### 1-6. 자료실 관리
> 클래스 소유자 여부와 관계없이 관리 가능

- 자료 **조회**
- 자료 **생성**
- 자료 **수정**
- 자료 **삭제**

---

## 2. 시스템 공통 기능

### 2-1. 인증
- 회원가입
- 로그인

---

### 2-2. 애로사항 게시판 (유저)
- 애로사항 게시글 **생성**
- 본인이 작성한 애로사항 게시글 **조회**
- 본인이 작성한 애로사항 게시글 **수정**
- 본인이 작성한 애로사항 게시글 **삭제**

---

## 3. 클래스 (Class)

### 3-1. 공통
- 클래스 내 기능 사용 가능
    - 강의
    - QnA 게시판
    - 자료실

---

### 3-2. 교수자
- 클래스 **생성**
    - 입력 데이터
        1. 클래스 제목
        2. 클래스 설명
        3. 교수자 (자동 생성)
        4. 클래스 개설 날짜 (자동 생성)
- 클래스 **수정**
- 클래스 **삭제**
    - 클래스 삭제 시 하위 데이터도 함께 삭제
        - 강의
        - QnA 게시판
        - 자료실
- 클래스 **관리 권한**
    - 클래스 소유자
    - 관리자

---

### 3-3. 학습자
- 클래스 **조회**
- 클래스 **참여**
- 클래스 내 강의 **조회**

---

## 4. 강의 (Lecture)

### 4-1. 교수자
- 하나의 클래스에 **여러 강의 생성 가능**
- 강의 영상 **업로드**
    - YouTube 영상만 지원
    - 입력 데이터
        1. 강의 제목
        2. 강의 설명
        3. 영상 링크 (YouTube)
        4. 생성 날짜 (자동 생성)
- 강의 **수정**
- 강의 **삭제**
- 강의 **관리 권한**
    - 클래스 소유자
    - 관리자
- 모든 학습자의 **강의 시청 여부 확인**

---

### 4-2. 학습자
- 강의 **조회**
- 강의 **시청**
- 본인의 강의 **시청 여부 확인**

---

## 5. 자료실 (Resource)

### 5-1. 교수자
- 자료 **업로드**
    - 입력 데이터
        1. 자료 제목
        2. 자료 설명
        3. 자료 파일
        4. 생성 날짜 (자동 생성)
- 자료 **수정**
- 자료 **삭제**

---

### 5-2. 학습자
- 자료 **조회**

---

## 6. QnA 게시판

### 6-1. 공통
- 클래스 참여자는 게시글 **작성 가능**
    - 입력 데이터
        1. 게시글 제목
        2. 게시글 내용
        3. 게시자 (자동 생성)
        4. 생성 날짜 (자동 생성)
- 본인 게시글 **수정**
- 본인 게시글 **삭제**
- 전체 게시글 **조회**

---

### 6-2. 교수자
- 게시글 소유자와 관계없이 **삭제 가능**
