# 클래스 수강 신청 및 관리 구현 상세

## 개요
기존의 오픈된 수강 신청 방식을 폐지하고, 코드를 통한 가입 및 관리자/교수자에 의한 수동 수강생 픽업(Pick-up) 방식으로 수강 모델을 개편하였습니다.

## 주요 구현 기능

### 1. 참여 코드 (Enrollment Code) 자동 발급
- **로직**: 클래스 생성 시 자동으로 6자리의 영문/숫자 혼합 조합 문자열 생성
- **저장 위치**: 클래스 데이터 스키마 내 `enrollmentCode` 필드
- **UI 반영**: 교수자 및 관리자 대시보드의 클래스 상세 뷰(`<header>` 우측)에 참여 코드를 상시 표출하여 외부 공유가 용이하게 함.

### 2. 코드를 통한 수강 신청 (학습자)
- **접근 경로**: 학생 대시보드 (`/student/page.js`) 내 "코드로 참여" 탭
- **입력 폼**: 6자리 참여 코드 입력
- **검증 로직**: `enrollments.joinWithCode(code, studentId)` API 내부에서 코스트와 매칭되는 클래스를 검색하고 존재 시 수강 목록에 추가.
- **UI/UX 개선 [NEW]**: '내 학습 공간'의 클래스 카드에 해당 클래스의 **담당 교수자 성함**을 명시하여 학습자가 소속 정보를 직관적으로 인지할 수 있도록 보강.

### 3. 수강생 수동 등록 및 검색 (관리자/교수자)
- **접근 경로**: 클래스 상세 뷰의 "수강생 관리" 탭 화면 내 `[+ 수동 등록]` 버튼
- **기능 플로우**:
  1. 관리자/교수가 학생의 이름 또는 ID 일부를 입력하여 검색 (`api.users.search`)
  2. 일치하는 시스템 내 가입자 목록을 리스트 형태로 모달창에 출력
  3. 특정 학생의 `[등록]` 버튼을 클릭하여 즉각 수강생으로 편입
- **장점**: 수강 코드를 직접 받지 못한 학생 구제 및 오프라인 명단을 바탕으로 한 시스템 일괄 등록의 편의 제공

### 4. 강의 등록 시 알림 연동 [NEW]
- **로직**: 교수자나 관리자가 새 강의를 등록(`api.lectures.create`)하면, 해당 클래스에 수강 등록된 모든 학생의 계정으로 `NEW_LECTURE` 타입의 알림이 생성됩니다.
- **효과**: 학생은 로그인 시 헤더의 알림 배지를 통해 새 학습 콘텐츠의 업로드 사실을 즉각 인지할 수 있습니다.
