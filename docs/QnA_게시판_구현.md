# 질문 답변(QnA) 게시판 시스템 상세 구현

## 개요
각 클래스 내부에서 해당 수강생 또는 담당 교수자가 쌍방 소통할 수 있는 QnA 게시판 컴포넌트를 구현했습니다.

## 주요 구현 사항

- **모듈화된 구조 [UPGRADE]**: 가독성과 유지보수 편의를 위해 기능을 세분화하여 `src/components/ui/qna/` 폴더에 분산 배치했습니다.
  - `QnaList.jsx`: 전체 목록 렌더링 및 모드(List, Detail, Write) 전환 총괄
  - `QnaDetailView.jsx`: 특정 질문 본문 및 답변 목록, 답변 등록 폼 담당
  - `QnaWriteView.jsx`: 새 질문 작성 폼 전용
- **UI 전환 전략**: 이전의 아코디언 확장 방식에서 **상태 기반 모드 전환(State-based View Switching)** 방식으로 변경되어, 상세 내용이나 글쓰기 시 전체 영역을 활용하여 사용자 몰입도를 높였습니다.

### 2. 권한별 쓰기 및 제어 로직
- **글 쓰기 (Create)**
  - 수강생(Student), 교수자(Prof), 관리자(Admin) 누구나 작성 가능하며, 작성 시 `isPrivate` 옵션을 통해 비공개 여부를 설정할 수 있습니다.
- **답변하기 (Reply) [NEW]**
  - **교수자/관리자 전용**: 학생의 질문에 대해 답변을 달 수 있는 전용 인터페이스가 제공됩니다.
  - 답변 데이터는 `replies` 배열로 관리되며, 각 답변은 작성자 정보와 타임스탬프를 포함합니다.
- **비공개 게시물 가시성 정책 [UPDATE]**
  - **이전**: 비공개 글은 제3자(다른 학생)에게 목록에서 완전히 숨김 처리됨.
  - **현재**: 모든 사용자가 목록에서 비공개 글의 제목, 작성자, 답변 여부를 확인할 수 있으나, 본인이 아닌 경우 상세 조회(클릭)는 차단됩니다. (🔒 아이콘 표시)
- **글 삭제 (Delete)**
  - **작성자 본인**: 본인의 질문 또는 본인의 답변 삭제 가능.
  - **관리자/교수자**: 모든 질문과 답변에 대해 강제 삭제 권한 보유.

### 3. UI 및 사용성 개선
- **전체 너비 카드**: 가독성을 위해 QnA 리스트 아이템을 전체 너비(100%) 카드 디자인으로 개편.
- **아코디언 확장**: 클릭 시 상세 내용과 답변 목록이 슬라이드 다운되는 직관적인 UX 제공.
- **상태 표시**: '답변 대기중', '답변 N개' 등의 상태를 색상과 함께 표시하여 처리 현황을 한눈에 파악 가능.

### 4. 알림 및 활동 이력 연동 [UPGRADE]
- **활동 로그**: QnA 생성, 답변 등록, 삭제 등의 모든 이벤트는 `api.logs` 서비스와 연동되어 실시간 기록됩니다.
- **푸시형 알림 (UI)**:
  - **새 질문 알림**: 학생이 질문을 등록하면 담당 교수자에게 알림이 전송됩니다.
  - **답변 완료 알림**: 교수자가 답변을 등록하면 질문 작성자에게 알림이 전송됩니다.
  - **공통**: 알림 클릭 시 해당 QnA가 위치한 클래스의 탭으로 즉시 이동합니다.
