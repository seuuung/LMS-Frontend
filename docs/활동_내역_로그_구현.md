# 활동 내역 (Activity Log) 시스템 구현 상세

## 개요
단순히 클래스 개설일만 표기되던 기존 방식에서 벗어나, 클래스와 관련된 모든 데이터(강의, 자료, 질문, 수강 등)의 CRUD 활동을 통합 기록하고 추적할 수 있는 활동 로깅 시스템을 구축했습니다.

## 주요 아키텍처 및 구현

### 1. 통합 로깅 허브 구축 (`api.logs`)
- 모든 상태 변경이 일어날 때마다 `api_mock.js` 내부의 `createLog()` 함수를 호출.
- **상세 기록 [UPDATE]**: 단순히 액션 이름만 남기는 것이 아니라, **'어떤 필드가 무엇에서 무엇으로 변경되었는지'** 혹은 **'어떤 구체적인 리소스(강의 제목 등)가 대상인지'** 메시지에 포함하여 가독성을 높임.
- 인자 구성: 액션(CREATE, UPDATE, DELETE), 엔티티 타입, 엔티티 ID, 액션 메시지, 발생자(Actor), **부모 클래스 식별자(ClassId)**.

### 2. 최신 변경사항 실시간 감지 UI
- 클래스 카드(Prof/Admin)에 기존 **[개설일]** 외에 **[최근 수정일]** 일자를 명확히 분리하여 표시.
- 해당 일자는 `logs` 데이터 중 `classId`가 대상 클래스인 가장 최신 항목의 타임스탬프를 가져와 표시.
- 하위 강의를 업로드하거나, 자료를 편집, 삭제하는 등 관련 로그가 기록될 경우 클래스 카드 자체의 최신활동 날짜가 즉각 변경됨.

### 3. 전체 로그 조회 모달 창 (`ActivityLogModal`)
- 각 클래스, 유저 카드 등의 `[로그 보기]` 버튼을 통해 접근.
- **로그 정렬**: 발생 시간 기준 내림차순(최신순) 정렬.
- **날짜 정보 분리**: 각 로그 항목마다 최초 생성 시각과 최종 수정 시각을 구분하여 표시할 수 있는 구조적 토대 마련.
