# 클래스 기본 관리 (CRUD) 구현 상세

## 개요
플랫폼의 핵심 거점이 되는 클래스(과목/강좌의 집합)의 개설에서부터 기본 정보 관리 절차를 구조화했습니다.

## 주요 구현 사항

### 1. 클래스 생성 (Create)
- **권한**: 교수자(Professor) 및 관리자(Admin)
- **제공 컴포넌트**: 교수자 대시보드 내 '새 클래스 개설' 모달, 관리자 창에서의 폼.
- **연동 데이터**: `title` (과목명), `description` (설명, 선택사항). _(추가로 개설 시 `enrollmentCode`가 발급되며 `profId`가 바인딩 됨)_

### 2. 클래스 목록 조회 (Read)
- **관리자 뷰**: 전체 데이터베이스에 존재하는 모든 클래스 리스트 및 상세 정보(`profId` 매칭 후 실명 노출) 조회 가능.
- **교수자 뷰**: 데이터베이스 전체가 아닌 `profId === currentUser.id` 조건에 해당하는, 본인이 개설한 클래스만 노출.
- **학습자 뷰**: 현재 수강 중(`enrollments` 연동 여부 기준)인 클래스 카드만 노출되도록 제한하여 직관성 확보.

### 3. 클래스 정보 수정 (Update)
- 모달창 기반 정보 수정(`Modal` 재사용) 
- 클래스의 타이틀과 설명 정보를 API(`api.classes.update`)를 통해 언제든 수정 가능.
- **UI 접근성**: 클래스 대시보드 진입 후 첫 탭인 '클래스 소개' 내부의 **강좌 소개 우측 연필 아이콘**을 클릭하거나, 클래스 목록 카드의 우측 상단 햄버거(⋮) 메뉴에서 **강의정보수정**을 선택하여 모달에 접근합니다.

### 4. 클래스 삭제 (Delete)
- 삭제 트리거(`api.classes.delete`) 시 클래스 본체 데이터 파기뿐만 아니라, **종속된 하위 엔티티들을 일괄 삭제(Cascade Effect)** 처리.
- 함께 삭제되는 데이터: 해당 클래스의 소속 강의(`lectures`), QnA 리스트(`qnas`), 관련된 학습 자료(`resources`), 사용자별 수강 목록(`enrollments`)

### 5. UI/UX 편의
- **Grid Layout**: 반응형 웹 기준 CSS `display: grid` 를 적극 활용해 클래스 카드가 브라우저 너비에 따라 1xn 부터 mxn 배열로 부드럽게 조정됨.
- 최신 상태 확인이 쉽도록 생성 날짜 및 '최근 활동 내역' UI 지원.
- **클래스 카드 UI 간소화 및 직관적 액션**: 기존의 길고 나열된 버튼들을 카드 우측 상단 햄버거(⋮) 메뉴(드롭다운 형태)로 통합하여 여백을 확보했습니다.
- 카드 바디의 여백이나 제목 텍스트 등을 직접 클릭해도, 해당 클래스의 관리(상세 대시보드) 페이지로 화면이 전환되도록 사용자 경험을 강화했습니다.
