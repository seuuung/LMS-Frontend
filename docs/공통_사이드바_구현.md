# 공통 사이드바 구현 상세

## 개요
각 사용자 권한(관리자, 교수자, 학습자)에 따른 사이드바 UI가 파편화되어 있던 것을 모듈화하여, 대시보드 및 하위 페이지(클래스 상세 등) 어디서나 일관된 네비게이션 경험을 제공하도록 개선했습니다.

## 주요 구현 사항

### 1. 전역 컴포넌트화 (`Sidebar.jsx`)
- 기존에 각 대시보드 파일 내부에 코딩되어 있던 네비게이션 구조를 독립된 컴포넌트로 분리.
- 컴포넌트 내부에서 `userRole` 값(admin, prof, student)을 읽어들여 해당 권한에 맞는 메뉴들만 조건부로 렌더링.

### 2. 레이아웃 안정화 (고정/스크롤 분리)
- 전체 레이아웃을 크게 좌측 Sidebar영역과 우측 Main Contents영역으로 분리
- 사이드바는 화면 스크롤 시에도 뷰포트 좌측에 고정 유지(`position: sticky` 또는 레이아웃 내 고정 너비 할당)되며 콘텐츠 영역만 단독으로 스크롤되는 구조 마련.

### 3. 상태 유지 및 라우팅 전략 ([UPGRADE])
- **쿼리 기반 탭 전환**: 대시보드 내 탭 전환 시 URL `searchParams`(예: `?tab=myClasses`)를 활용하여 브라우저의 뒤로가기/앞으로 가기 연동을 지원합니다.
- **라우팅 최적화 (`handleMenuClick`)**:
  - **Push**: 현재 페이지가 대시보드 외부(예: 강의 시청 페이지)일 경우, 대시보드 메인으로 이동하기 위해 `router.push`를 사용합니다.
  - **Replace**: 이미 대시보드 내에 있는 경우, 브라우저 뒤로가기 히스토리를 쌓지 않기 위해 `router.replace`를 사용하여 쿼리 스트링만 즉시 교체합니다.
